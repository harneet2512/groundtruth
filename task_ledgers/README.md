# task_ledgers/

One ledger per SWE-bench task audited under the **gt_trial.md ¬ß4** protocol. Each ledger is the
per-gt_gt-COMPONENT audit of that task's trajectory ‚ÄĒ **REAL VALUES, `GT SENT` ‚ÜĒ `AGENT DID`,
read (never grep) from `output.jsonl`**. Mandated by `CLAUDE.md` + `gt_trial.md ¬ß4`.

## Format (per gt_trial.md ¬ß4) ‚ÄĒ every ledger contains, per task:
- **PREREQS table** ‚ÄĒ substrate P1 resolution / P2 graph.db / P3 embedder, the **8-dp real values**
  verbatim from the gate-deep JSON, `GREEN?`, and **how it reached the agent** (substrate numbers
  are telemetry-only ‚Üí they reach the agent ONLY as the brief's resolved-edge lines).
- **One table PER COMPONENT** (L1 ¬∑ L3b ¬∑ consensus ¬∑ L3/GT_VERIFY ¬∑ L4 ¬∑ L5 ¬∑ L5b ¬∑ L6), columns:
  `turn | GT SENT (verbatim bytes the agent saw) | AGENT DID (verbatim action at/after) | D/C/C`.
- **per-table verdict** (D/C/C + leakage count) + a **cross-component line** (leakage MUST be 0;
  consumed-count; fair-probe-count).

**Append-only across runs** ‚ÄĒ never overwrite a prior run's task ledger; add the new run's audit
section under a dated heading.
